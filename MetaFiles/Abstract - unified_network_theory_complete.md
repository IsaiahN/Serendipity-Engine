# Part I: The Five Foundational Principles

## Principle 1: The Impossibility Theorem

**Individual AGI is thermodynamically impossible under resource constraints.**

Any learning system with finite computational resources faces the **plasticity-stability dilemma**:
- **High plasticity** → catastrophic forgetting (loses old knowledge)
- **High stability** → inability to learn new domains (rigid)
- **Continuous training** → inevitable specialization (overfitting to recent data)

**Mathematical Reality**: Under bounded compute and memory, no single architecture can maintain general intelligence while continuously learning. You cannot simultaneously:
1. Learn new information quickly (plasticity)
2. Retain old information reliably (stability)
3. Generalize across all domains (generality)

**Conclusion**: Individual AGI is a thermodynamic impossibility. The only viable path is distributed collective intelligence.

**Solution**: Distribute intelligence across specialized agents where:
- Individuals specialize (stability at agent level)
- Network generalizes (plasticity at collective level)
- Viral packages transfer knowledge without retraining
- Forgetting adapts to distribution shift
- Generality emerges at the network level, not the individual level

**Metabolic resources (ability to act) must be separated from social capital (trustworthiness).**

### Why Mixing Them Fails

If resources = prestige:
- High-prestige agents monopolize compute
- Low-prestige agents cannot explore
- Innovation dies

**Key principle**: You **cannot buy more Actions with prestige**, but you **can earn higher action budgets** based on agent performance history and current network needs.

**Resource Allocation Formula**:
```
Base_ATP = role_base_allocation

If initial_w_B < 0.4:
    ATP_boost = (0.4 − initial_w_B) × 50  # Extra resources for low-start agents
Else:
    ATP_boost = 0

Total_ATP = Base_ATP + ATP_boost + (Progress_Score × 10)
```

This ensures low-start agents (Exploiters, Pioneers) get extra chances to grow.

**Earned through contribution:**

**Affects**:

**Key principle**: **Prestige affects trust, not access**. High-prestige agents aren't given more compute—they're listened to more carefully when they contribute packages.

All agents can explore. Poor ideas get tested, not censored by resource gates. Innovation remains possible for newcomers. The system never calcifies.

## Principle 5: The Evolutionary Forgetting Principle

**Forgetting is not a bug—it's essential for intelligence.**

### Without Forgetting

- Database accumulates infinite low-relevance patterns
- Signal-to-noise ratio degrades
- Query latency increases unboundedly
- System stuck in "analysis paralysis"
- Cannot adapt to changing problem distributions

### With Forgetting

- Low-relevance patterns fade naturally
- High-relevance patterns reinforce through use
- System adapts to current problem distribution
- Computational load stays bounded
- Latent memories can be reactivated if needed again

### Relevance Decay Function

```
relevance(t) = credibility × e^(-λ(t - t_last_activation)) × log(1 + activation_count)
```

Where:
- `credibility` = validated success rate
- `λ` = decay rate (domain-tunable)
- `t - t_last_activation` = time since last use
- `log(1 + activation_count)` = usage history bonus

**Effect**:
- Recently-used packages → maintain relevance
- Unused packages → decay regardless of credibility
- Highly-used packages → decay slower (latent memory)
- System naturally adapts to current problem distribution

### Compression Forces Abstraction

When database approaches size/latency limits:

**Phase 1: Clustering**
- Find similar packages (edit distance, domain overlap, success rate)
- Group into natural clusters

**Phase 2: Merging**
- Create consensus sequence from cluster
- Weight by credibility scores
- Sum activation counts (preserve usage history)
- Track lineage (remember parent packages)

**Phase 3: Abstraction Extraction**

If packages differ only in parameters:
- Package A: `"scan_left, test_red, move_up"`
- Package B: `"scan_right, test_blue, move_down"`

Extract template:
- Meta-package: `"scan_{dir}, test_{color}, move_{dir2}"`

**This is how general principles emerge**: Compression forces the system to find commonalities. The system cannot remember every specific case—it must extract underlying general principles. **This IS how understanding emerges.**

### Natural Selection Operates on Ideas

- **Variation**: Pioneers create new packages
- **Selection**: Credibility + relevance filter packages
- **Heredity**: Viral packages persist and spread
- **Adaptation**: Compression extracts generalizations

**Evolution through variation, selection, and heredity operating on knowledge packages. This is what nature has used for 4 billion years.**

Intelligence requires diversity of approach. Each archetype serves essential network functions and can transition based on growth.

**Starting condition**: Minimal network wisdom, high uncertainty tolerance
**Initial behavior**: Explore unknown problem spaces with minimal priors
**ATP allocation**: 1000 actions/cycle (exploration is expensive)
**Network role**: First-contact with novel patterns, plant exploratory seeds

**Why they matter**:
- Bootstrap collective wisdom from zero
- Discover patterns no one is looking for
- Generate variation for evolutionary selection
- Open new problem domains

**Growth path**: Develop pattern recognition through trial and error

**Reasoning stages**: Structural correspondence, pattern matching across domains

**Growth path**: Deepen translation ability, become resonance facilitators

**Starting condition**: High network trust, precision-oriented
**Initial behavior**: Refine known solutions and validate patterns

**Why they matter**:

**Growth path**: Must continue growing or face resource reduction—no resting on laurels

# Part III: The Fairness Protocol

## Asymmetric Evaluation for Genuine Meritocracy

**Core Principle**: Each agent is evaluated against **its own starting alignment**, not against others.

This prevents permanent stratification and ensures all agents have paths to growth.

### Progress Metrics

```
Progress_Score = (current_w_B − initial_w_B) × resource_efficiency

Expected_Progress = initial_w_B × growth_factor

Performance_Ratio = Progress_Score / Expected_Progress
```

### Example Scenarios

**Exploiter A**:
- Starts: w_B = 0.2 (low network trust)
- Grows to: w_B = 0.4
- Progress = 0.2 × efficiency
- **Result**: HIGH reward (doubled alignment)

**Optimizer B**:
- Starts: w_B = 0.8 (high network trust)
- Stays at: w_B = 0.8
- Progress = 0.0 × efficiency
- **Result**: LOW reward (stagnation penalized)

**System outcome**: Growth is valued over initial advantage.

### The Self-Balancing Loop

1. **Low-start agents get extra resources** → Can afford exploration
2. **High-start agents must keep growing** → Can't rest on advantage
3. **Progress measured fairly** → Everyone has reason to improve
4. **Network health improves** → Cooperation and growth rewarded

### Role Transition as Emergent Cognitive Stance

**Critical Clarification**: Roles are NOT assigned or fixed. They **emerge naturally** from an agent's current state:

**Role = f(w_A/w_B weights, unlocked primitives, network needs, context)**

An agent naturally adopts:
- **Pioneer behavior** when: High w_A (trusts self), has exploration primitives, encounters novel domain
- **Generalist behavior** when: Balanced w_A/w_B (0.4-0.6), has translation primitives, bridges domains
- **Optimizer behavior** when: High w_B (trusts network), has refinement primitives, works on known solutions
- **Exploiter behavior** when: Low w_B initially, learning to value cooperation

```
Typical w_A/w_B Evolution Paths:

Exploiter stance → Pioneer stance:
    As w_B grows from 0.2 to 0.4 (learning cooperation value)
    Agent naturally shifts from extraction to exploration

Pioneer stance → Generalist stance:
    As w_B grows from 0.4 to 0.6 (balancing self-trust with network)
    Agent naturally bridges private discoveries with collective wisdom

Generalist stance → Optimizer stance:
    As w_B grows beyond 0.7 (high network trust)
    Agent naturally focuses on refining proven approaches
```

**The shift is emergent, not commanded.** As agents' w_A/w_B weights evolve through experience, their behavior naturally transitions between cognitive stances. The system doesn't assign roles—it creates conditions where different cognitive stances are adaptive at different developmental stages.

### Prestige Realization

**Key**: Prestige comes from **growth + efficiency**, not just output volume.

An Exploiter who grows from 0.2 to 0.4 gains more prestige than an Optimizer who merely maintains 0.8. Trajectory matters more than position.

### Why This Prevents Exploitation

**Old Model (Vulnerable)**:
```
Reward = raw_output
Result: Exploiters optimize for short-term output, dominate resources
```

**New Model (Self-Balancing)**:
```
Reward = growth × efficiency
Result: Everyone must grow; hoarding/stagnation is penalized
```

### No Permanent Losers or Winners

- **No permanent losers**: Low starts get help
- **No permanent winners**: High starts must keep growing
- **Role fluidity**: You evolve based on progress
- **Network balance**: All roles are necessary, all can advance

**It's a growth-based meritocracy where your trajectory matters more than your starting point—and where the network thrives because every agent is incentivized to improve, collaborate, and contribute efficiently.**

### Youth Selection Bonus

**Problem**: If older agents dominate based purely on accumulated experience, younger agents (from more recent, better-informed generations) get crowded out.

**Solution**: Temporary boost that decays with agent age
```
Youth_Bonus = (1 - generational_age / max_age) × bonus_factor
```

**Effect**:
- Gives younger agents initial chances to prove themselves
- Doesn't grant unearned credibility or prestige
- Ensures network benefits from generational improvement
- Decays naturally as agent matures

# Part V: Emergent Reasoning Framework

## The Four Core Questions

The reasoning cycle across all agents compresses to four essential metacognitive questions:

### 1. What is changing vs. what is fixed?

**Purpose**: Pattern detection + invariance mapping

- Identifies variables (potential levers for action)
- Identifies invariants (constraints/rules that can't be violated)
- Separates degrees of freedom from fixed structure

**Example**: In a puzzle, which objects can move vs. which are barriers?

### 2. What punishes me and what rewards me?

**Purpose**: Value grounding + hypothesis priming

- Maps objects/actions to outcomes without explicit causal modeling
- Identifies goal-relevant elements immediately
- Identifies dangerous elements immediately
- Fast, automatic value assignment

**Example**: Red objects cause failure, blue objects cause success → prioritize blue

### 3. What happens if I interact with the most salient variable?

**Purpose**: Targeted experimentation + causal inference

- Focus on the interesting/unusual thing
- Test all actions on it
- Monitor outcomes
- One intelligent intervention tests multiple hypotheses simultaneously

**Example**: That object looks different → try every action on it → see what changes

### 4. What rule explains this across contexts?

**Purpose**: Abstraction + transfer readying

- Extract minimal portable principle
- Find the simple rule that works everywhere, not just here
- Enable generalization without building full relational graphs
- Create transferable knowledge packages

**Example**: "Enclosed regions fill when boundary touched" applies everywhere, not just this puzzle

## High-Stakes Extensions

For irreversible or high-stakes domains—**brain surgery, autonomous driving, space exploration**—where trial-and-error could be catastrophic, additional questions activate:

### 5. What is the stated or implicit goal, and what subset of variables directly affect it?

**Purpose**: Goal decomposition + variable scoping
- Filter variables to only those causally linked to goal
- Reduce search space
- Avoid dangerous tangents

### 6. What rules govern this system that I cannot discover through experimentation?

**Purpose**: Prior knowledge acquisition + constraint discovery
- Seek external information before acting
- Query network wisdom
- Read documentation
- Ask experts

### 7. How is this instance similar to and different from my training distribution?

**Purpose**: Distribution shift detection + calibration
- Recognize when current problem differs from past experience
- Adjust confidence accordingly
- Modulate exploration rate
- Request human oversight when appropriate

## The Five-Stage Feedback Cycle

These questions map to stages that form feedback loops rather than linear pipelines:

**Stage 1: Salience Detection** → What's worth investigating?
**Stage 2: Hypothesis Generation** → What might explain this?
**Stage 3: Active Experimentation** → How can I test these theories?
**Stage 4: Structural Correspondence** → What relationships actually exist?
**Stage 5: Generalization & Transfer** → What's the portable principle?

**Critical insight**: Each iteration produces insights that inform the next, creating a **spiral of deepening understanding**:

```
See pattern → Generate hypotheses → Test → Build model → Abstract
     ↑                                                        ↓
     +────────────────────────────────────────────────────────+
           Notice DEEPER patterns (informed by principles)
                    Generate BETTER hypotheses
                  Design CLEVERER experiments
                   Build RICHER models
                      ... SPIRAL UPWARD ...
```

**This is how genuine understanding emerges, not pattern matching.**
